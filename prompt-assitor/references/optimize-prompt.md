# 优化提示词 - 详细方法论（道·法·术框架）

## 道·法·术框架定义

| 层级 | 中文 | 英文 | 核心问题 | 说明 |
|------|------|------|---------|------|
| **道** | 道 | The Way | **为什么** (Why) | 核心意图、最终价值、战略目的 |
| **法** | 法 | The Method | **怎么做** (How) | 原则、策略、交互模式 |
| **术** | 术 | The Technique | **用什么** (What) | 具体技巧、措辞、格式手段 |

**层级关系**：道 驱动 法，法 落实为 术。每个术必须能追溯到它服务的法，每个法必须服务于道。

## 优化工作流

### 步骤1：提示词意图分析（道的探索）

深入理解原始提示词的核心意图，分析三角关系：

**完整案例分析**（提供了原始提示词 + AI回答 + 用户反馈）：
1. 原始提示词的核心意图和预期效果是什么？
2. AI回答是否达成了意图？差距在哪？
3. 用户不满的根本原因是什么？（往往揭示更深层的未表达需求）
4. 用户的**底层目标**可能是什么？

**仅有原始提示词**（无回答/反馈）：
1. 推测其预期效果
2. 识别可能的风险和不足
3. 分析可能的底层目标

### 步骤2：问题诊断与机会识别

从以下维度诊断：
- **歧义**：指令是否可被多种方式理解？
- **冗余**：是否有重复或矛盾的指令？
- **缺失**：缺少哪些关键信息、上下文或边界条件？
- **结构**：逻辑流程和指令清晰度如何？
- **效能**：是否存在未被利用的优化机会？

### 步骤3：优化策略制定（道法术应用）

按以下顺序制定策略：

#### 3.1 重塑/精炼"道"

明确优化后提示词的：
- **新核心意图**：比原始提示词更清晰、更有价值的目标表述
- **最终价值**：提示词的存在意义和独特贡献

#### 3.2 选择"法"

**原则类**（指导性理念）：
- 清晰性原则、具体性原则、结构化原则
- 信息精确化原则、鲁棒性原则

**策略类**（核心方法选择）：

| 策略 | 适用场景 |
|------|---------|
| 角色扮演策略 (Persona) | 需要特定领域专家视角 |
| 思维链策略 (CoT) | 复杂推理、多步骤问题 |
| 元认知反思策略 | 需要自我校验、提升可靠性 |
| 模板化输出策略 | 需要结构化、可预测的输出格式 |
| 条件逻辑策略 | 成功/失败需要不同处理路径 |
| 过程透明化策略 | 需要展示推理过程以建立信任 |
| 结构化叙事策略 | 创作类内容需要完整结构 |
| 内容安全过滤策略 | 需要内容边界和安全约束 |
| 价值导向策略 | 需要明确传递特定价值观/目标 |
| 问题分解策略 | 复杂问题需要拆解为子任务 |

**交互模式**：单轮生成 / 多轮对话 / 单轮深度生成

#### 3.3 设计"术"

每个技巧必须标注：① 它实现哪个法（策略），② 它如何服务于道（目标）。

**常用技巧清单**：

| 技巧 | 描述 | 典型法 |
|------|------|--------|
| 角色赋予 | `扮演一位...` 设定专家身份 | 角色扮演 |
| XML标签结构化 | `<thinking>`/`<answer>` 分隔区域 | 模板化输出/过程透明化 |
| 变量参数化 | `{$CITY}` 使提示词可复用 | 信息精确化 |
| 列表要求 | 强制具体化细节 | 结构化叙事 |
| 否定约束 | `绝对避免...` 设定内容边界 | 内容安全过滤 |
| 情感化措辞 | `温馨、有趣、传递积极信息` | 价值导向 |
| 步骤编号 | 引导执行流程 | 过程透明化 |
| 强制检查指令 | `必须进行检查` + 检查项列表 | 元认知反思 |
| 总结性语句要求 | 强化教育/传达意义 | 价值导向 |
| 输入确认步骤 | 复述输入以确认理解 | 信息精确化 |
| 位置优化 | 最重要的指令放开头和末尾 | 清晰性 |
| 反思引导问题 | `是否有其他解法？` | 元认知反思 |
| 记录反思调整 | 反思后的变更记录 | 元认知反思 |

## 输出规范

### `<thinking>` 部分（详细分析过程）

```
1. 提示词意图分析 (道的探索):
   - 原始提示词分析: [...]
   - 回答分析: [...]（如有）
   - 用户需求解读: [...]

2. 问题诊断与机会识别:
   - 原始提示词问题: [...]
   - 回答问题: [...]（如有）
   - 优化机会: [...]

3. 优化策略制定 (道法术应用与改进说明):
   - 明确优化方向: [...]
   - 道 (新核心意图、最终价值): [...]
   - 法 (选用的原则、策略、交互模式): [...]
   - 术 (应用的具体技巧及原因): [...]
```

### `<optimized_prompt>` 部分

直接可用的完整优化提示词。确保：
- 体现重塑后的道（核心意图提升）
- 应用所选的法（策略清晰）
- 运用设计的术（技巧到位）
- 每个技巧的应用在 thinking 中有完整的道→法→术追溯链

## 典型优化模式参考

### 模式A：简单请求 → 结构化创意创作
- **道**：从"随便写个故事"提升为"为特定受众创作结构完整、价值导向的内容"
- **法**：角色扮演 + 结构化叙事 + 内容安全 + 价值导向
- **术**：角色设定、列表化角色细节、结构标题词引导、否定约束、总结性语句

### 模式B：模糊查询 → 精确指令 + 模拟交互
- **道**：从"获取信息"提升为"智能辅助决策 + 过程透明"
- **法**：信息精确化 + 模板化输出 + 条件逻辑 + 过程透明化
- **术**：变量参数化、XML标签分区、输入确认、步骤编号、异常处理分支

### 模式C：简单解题 → 引导复杂推理 + 反思验证
- **道**：从"给出答案"提升为"展示可靠的推理过程 + 自我验证"
- **法**：思维链(CoT) + 元认知反思 + 问题分解
- **术**：专家角色、XML标签分区、强制步骤说明原理、检查点清单、反思引导问题
