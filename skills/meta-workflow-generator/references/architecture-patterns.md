# 工作流架构模式参考

## 通用架构模式

### 模式 A：线性流水线

最简单的模式，适用于环节之间有严格先后依赖的任务。

```
[澄清] → [策略] → [生成] → [审查] → [交付]
```

**适用领域：** 内容创作、报告撰写、方案设计
**特征：** 每个环节的输出是下一个环节的唯一输入，无分支、无并行。

---

### 模式 B：扇出-汇聚

中间环节按维度拆分为多个并行技能，最后汇聚整合。

```
[澄清] → [策略] ─┬→ [生成-维度A]
                   ├→ [生成-维度B] ──→ [整合] → [审查] → [交付]
                   └→ [生成-维度C]
```

**适用领域：** 多维度分析（财务+运营+市场）、全栈设计（前端+后端+数据）、多渠道营销
**特征：** 并行环节之间无依赖，可由不同专业技能独立完成，最终汇聚为完整交付物。

---

### 模式 C：迭代循环

核心生成环节与审查环节形成反馈循环，直到质量达标。

```
[澄清] → [策略] → [生成] ⇄ [审查] → [交付]
                      ↑        ↓
                      └── [修正] ←┘
```

**适用领域：** 创意工作（广告文案、品牌策略）、学术写作、代码审查
**特征：** 审查结果可能触发重新生成或修正，需要设定最大迭代次数（建议 3 轮）。

---

### 模式 D：阶段门禁

每个阶段结束时有质量门禁，通过后才能进入下一阶段。

```
[澄清] →|门禁|→ [设计] →|门禁|→ [执行] →|门禁|→ [验收]
```

**适用领域：** 项目管理、合规流程、教育课程设计
**特征：** 门禁定义了明确的通过标准，未通过则退回上一阶段修正。

---

### 模式 E：编排驱动

一个编排技能作为中枢，动态调度其他技能。

```
              ┌→ [技能A]
[编排器] ────┼→ [技能B]
              ├→ [技能C]
              └→ [技能D]
```

**适用领域：** 复杂项目（5+ 环节）、需要动态决策下一步的场景、多角色协作
**特征：** 编排器维护状态机，根据当前进度和条件决定调度顺序。

---

## 领域架构参考

### 内容营销领域

```
content1-topic-research    → 选题研究与定位
content2-outline-design    → 大纲与论证结构设计
content3-draft-writing     → 初稿撰写
content4-quality-review    → 多维质量审查（逻辑/文风/SEO/受众匹配）
content5-publish-optimize  → 发布优化（标题/摘要/标签/分发策略）
```

架构模式：线性流水线 + 迭代循环（draft ⇄ review）

---

### 数据分析领域

```
analyst1-question-framing   → 问题定义与假设建立
analyst2-data-exploration   → 数据探索与清洗策略
analyst3-analysis-execution → 分析执行与统计验证
analyst4-insight-synthesis  → 洞察提炼与故事化
analyst5-report-delivery    → 报告生成与可视化
```

架构模式：线性流水线 + 阶段门禁（每步验证假设）

---

### 产品运营领域

```
ops1-goal-alignment     → 目标对齐与指标拆解
ops2-strategy-design    → 策略设计与路径规划
ops3-resource-planning  → 资源规划与排期
ops4-execution-tracking → 执行追踪与异常预警
ops5-review-iteration   → 复盘与迭代优化
```

架构模式：阶段门禁 + 迭代循环（review → next cycle）

---

### 教育课程设计领域

```
edu1-learner-analysis    → 学习者分析与目标设定
edu2-content-structure   → 知识结构与学习路径设计
edu3-material-creation   → 教学材料创建（按模块并行）
edu4-assessment-design   → 评估与反馈机制设计
edu5-pilot-iteration     → 试讲/试学迭代优化
```

架构模式：扇出-汇聚（模块并行创建）+ 迭代循环（试讲反馈）

---

## 架构选择决策树

```
任务环节数量？
├── ≤ 4 个环节 → 模式 A（线性流水线）
├── 5-8 个环节 → 继续判断 ↓
│   ├── 有并行独立维度？ → 模式 B（扇出-汇聚）
│   ├── 需要反复打磨？ → 模式 C（迭代循环）
│   ├── 有严格质量关卡？ → 模式 D（阶段门禁）
│   └── 需要动态决策？ → 模式 E（编排驱动）
└── > 8 个环节 → 先合并/分层，再选模式
```

**组合使用：** 实际工作流通常是多种模式的组合。例如"线性流水线 + 局部迭代循环"是最常见的组合形式。
